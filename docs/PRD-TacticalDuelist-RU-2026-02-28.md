## 1. Обзор продукта

### 1.1 Название
**Tactical Duelist** — Mind Games Arena

### 1.2 В одну строку
Тактическая 1v1 игра с одновременными ходами, где ты программируешь действия своего героя и обыгрываешь противника через предсказание, позиционирование и mind games.

### 1.3 Видение
Создать эталонную «игру предсказаний» — скилловый соревновательный опыт, где главный кайф — это чтение мыслей противника. Никаких рефлексов, никакого гринда, никакого pay-to-win. Чистая стратегия и предсказание.

### 1.4 Проблема
Мобильный соревновательный гейминг поделён между рефлекс-играми (шутеры, MOBA) и играми с большой долей удачи (гача, авто-баттлеры). Нет мейнстримной мобильной игры, которая передаёт покерное «чтение противника» в быстром, визуальном, доступном формате.

### 1.5 Решение
1v1 тактическая игра, где оба игрока одновременно программируют ходы героя, а затем смотрят, как они исполняются в реальном времени. Каждое решение имеет вес — одно попадание убивает. Короткие матчи (3-5 минут). Глубокая стратегия через асимметричных героев с уникальными параметрами.

### 1.6 Ключевые метрики успеха
- **Ретеншн:** D1 > 40%, D7 > 20%, D30 > 10%
- **Длина сессии:** В среднем 15-20 минут (3-5 матчей)
- **Длина матча:** В среднем 3-5 минут
- **Матчей в день на игрока:** 5-10
- **Соревновательность:** Активная рейтинговая лестница с сезонными сбросами

### 1.7 Позиционирование
«Соревновательная игра предсказаний» — скилловая тактическая дуэль.

## 3. Основные игровые механики

### 3.1 Структура матча

```
ПОТОК МАТЧА:

  Выбор героя → Раунд 1 → [Исполнение] → Раунд 2 → [Исполнение] → Раунд 3 → Результат

  Раунды: до 3 (Первая кровь — первое убийство = победа)
  Шагов за раунд: зависит от героя (3-6)
  Время планирования: 30 секунд (раунд 1), 20 секунд (раунды 2-3)
  Условие победы: первый игрок, уничтоживший противника
  Ничья: если за 3 раунда никто не уничтожен
```

### 3.2 Фаза перед матчем

Оба игрока одновременно выбирают:
1. **Героя** — из разблокированного ростера
2. **Вариант способности** — разблокируется через Мастерство (будущая функция)
3. **Перк** — пассивный бонус (будущая функция)

После выбора обоих — выбор РАСКРЫВАЕТСЯ противнику. Mind games начинаются до первого шага.

**Критерии приёмки:**
- Экран выбора героя показывает всех героев со статами
- Таймер 15 секунд на выбор
- Если время истекло — последний выбранный герой фиксируется
- Оба героя + варианты раскрываются одновременно после фиксации обоих

### 3.3 Фаза планирования

Каждый раунд оба игрока программируют действия своего героя:

```
ИНТЕРФЕЙС ПЛАНИРОВАНИЯ:

  [Карта-сетка]     ← Текущее состояние карты
  [Позиция героя]   ← Ваш герой подсвечен
  [Очередь действий] ← 3-6 слотов (зависит от параметра ШАГИ героя)
  
  Доступные действия:
    ДВИЖЕНИЕ     — переместиться на 1-2 клетки вперёд (зависит от СКОРОСТИ)
    ПОВОРОТ_Л    — поворот на 90° влево
    ПОВОРОТ_П    — поворот на 90° вправо
    РАЗВОРОТ     — поворот на 180°
    ВЫСТРЕЛ      — стрельба в направлении взгляда (дальность зависит от героя)
    ОЖИДАНИЕ     — стоять на месте
    СПЕЦПРИЁМ    — уникальная способность героя (1 раз за раунд)
  
  Ограничения:
    - ВЫСТРЕЛ не может следовать за ВЫСТРЕЛОМ в пределах [ПЕРЕЗАРЯДКИ] шагов
    - СПЕЦПРИЁМ — 1 раз за раунд
    - Действия заполняются слева направо
    - Можно перетаскивать действия до фиксации
```

**Критерии приёмки:**
- Drag-and-drop программирование действий
- Визуальный превью: полупрозрачный путь показывает проецируемое движение
- Индикатор дальности стрельбы показывает опасную зону
- Таймер: 30с раунд 1, 20с раунды 2-3
- Кнопка «ГОТОВ» фиксирует действия досрочно (оба готовы = пропуск таймера)
- Если таймер истёк — оставшиеся пустые слоты заполняются ОЖИДАНИЕМ

### 3.4 Фаза исполнения — Пошаговое разрешение

Ядро технической системы. Действия обоих игроков исполняются ОДНОВРЕМЕННО, шаг за шагом.

```
ПОРЯДОК РАЗРЕШЕНИЯ (на каждом шаге):

  ФАЗА 1: ДВИЖЕНИЕ
    - Действия ДВИЖЕНИЯ/ПОВОРОТА обоих игроков исполняются одновременно
    - Игроки прибывают на новые позиции
    - Проверка столкновения: оба пришли на одну клетку → оба остаются на старых
    - Проверка подбора: если игрок встал на клетку с предметом → предмет подобран

  ФАЗА 2: СТРЕЛЬБА
    - ВЫСТРЕЛ/СПЕЦПРИЁМ обоих игроков исполняются одновременно
    - Выстрел: мгновенный луч из ТЕКУЩЕЙ позиции (после движения) в направлении взгляда
    - Луч летит на [ДАЛЬНОСТЬ] клеток
    - Проверка попадания: проходит ли луч через клетку, где СЕЙЧАС стоит противник?
    
  ФАЗА 3: УРОН
    - Если один игрок попал: 
        → У цели есть Броня: Броня уничтожена, цель выживает
        → У цели нет Брони: ЦЕЛЬ УНИЧТОЖЕНА → ПОБЕДА
    - Если оба попали друг в друга (взаимное попадание):
        → Оба выстрела ОТМЕНЯЮТСЯ (как ничья в «камень-ножницы-бумага»)
        → Никто не получает урон
        → Без исключений — взаимное попадание ВСЕГДА отменяется
    - Если никто не попал: переход к следующему шагу
```

#### 3.4.1 Детали движения

```
ДВИЖЕНИЕ:
  - Герой перемещается на [СКОРОСТЬ] клеток в направлении взгляда
  - СКОРОСТЬ 1: движение на 1 клетку
  - СКОРОСТЬ 2: движение на 2 клетки (проходит через промежуточную)
  - Нельзя двигаться сквозь стены
  - Нельзя выйти за край карты
  - Если путь заблокирован стеной: герой остаётся на месте (действие потрачено впустую)

ПОВОРОТЫ:
  - ПОВОРОТ_Л: вращение на 90° против часовой
  - ПОВОРОТ_П: вращение на 90° по часовой  
  - РАЗВОРОТ: вращение на 180°
  - Поворот НЕ перемещает героя
  - Герой остаётся на той же клетке

ОЖИДАНИЕ:
  - Герой стоит на месте, сохраняет направление взгляда
  - Стратегическое использование: выжидание перезарядки, ожидание пока противник 
    войдёт в зону поражения
```

#### 3.4.2 Детали стрельбы

```
ВЫСТРЕЛ:
  - Мгновенный луч в направлении взгляда
  - Длина луча = параметр ДАЛЬНОСТЬ героя
  - Луч блокируется стенами (останавливается у первой стены)
  - Исключение: Фазовый Выстрел Мага проходит через 1 стену
  - Исключение: Пробивной Ястреба проходит через ВСЕ стены
  
  ПЕРЕЗАРЯДКА:
    После ВЫСТРЕЛА герой не может СТРЕЛЯТЬ ещё [ПЕРЕЗАРЯДКА] шагов
    ПЗ 0 (Берсерк): может стрелять каждый шаг
    ПЗ 1 (Танк, Тень, Разведчик, Призрак, Мираж): стрельба через шаг
    ПЗ 2 (Лучник, Маг, Подрывник, Страж, Инженер): стрельба каждый 3-й шаг
    ПЗ 3 (Ястреб): один выстрел за раунд (фактически)

  ПРАВИЛО ВЗАИМНОГО ПОПАДАНИЯ:
    Если оба стреляют И оба попадают → выстрелы ОТМЕНЯЮТСЯ.
    Никто не получает урона. Как ничья в «камень-ножницы-бумага».
    Это создаёт стратегический слой: НЕ стреляй когда стреляет противник
    (если вы в дальности друг друга).
    Лучше двигаться и стрелять СЛЕДУЮЩИЙ шаг, когда у него перезарядка.
```

#### 3.4.4 Альтернативный сценарий — Уклонение

Что если бы Танк на шаге 3 двинулся ВБОК вместо вперёд?

```
═══ АЛЬТ. ШАГ 3 — ТАНК УКЛОНЯЕТСЯ ═══

  Лучник: ВЫСТРЕЛ ↑ (из столбец 4, ряд 7) [допустим, стрелял здесь вместо ОЖИДАНИЯ]
  Танк: ДВИЖЕНИЕ вбок (столбец 4 → столбец 5, ряд 2)

  Фаза 1 — Движение:
    Танк перемещается на (столбец 5, ряд 2). УШЁЛ с вертикальной линии!
    
  Фаза 2 — Стрельба:
    Лучник стреляет ВВЕРХ по столбцу 4: ряд7 → ряд6 → ... → ряд1.
    Танк на (столбец 5, ряд 2)? НЕТ — другой столбец.
    → ПРОМАХ!

  ★ Танк УКЛОНИЛСЯ боковым движением! ★
  Это выражение скилла: предскажи КОГДА противник стреляет → уйди с линии огня.
```

А если бы Танк отступил НАЗАД (от Лучника)?

```
═══ АЛЬТ. ШАГ 3 — ТАНК ОТСТУПАЕТ ═══

  Лучник: ВЫСТРЕЛ ↑ (из столбец 4, ряд 7)
  Танк: ДВИЖЕНИЕ назад/ВВЕРХ (ряд 2 → ряд 1)

  Фаза 1 — Движение:
    Танк перемещается на (столбец 4, ряд 1). Всё ещё на столбце 4!
    
  Фаза 2 — Стрельба:
    Лучник стреляет ВВЕРХ по столбцу 4: ряд7 → ... → ряд1.
    Танк на (столбец 4, ряд 1)? ДА — всё ещё на линии.
    Расстояние 6. Дальность Лучника 8. В пределах.
    → ПОПАДАНИЕ! Танк уничтожен.

  ★ Движение назад ПО ТОЙ ЖЕ ЛИНИИ не помогает! ★
  Нужно шагнуть ВБОК для уклонения. Это ядро пространственного мышления.
```

### 3.5 Фаза после раунда

После каждого раунда (если нет уничтожения):

1. **Исполнение**: Полная анимация действий обоих игроков. Оба видят, что делал противник.
2. **Сужение карты**: Внешнее кольцо клеток становится «опасной зоной» (мгновенное убийство если стоишь там в конце раунда). Визуально: красные/огненные клетки.
3. **Таймер нового раунда**: 20 секунд на программирование.

**Эскалация информации**: 
- Раунд 1: Ноль информации. Чистая интуиция.
- Раунд 2: Ты видел действия противника в Раунде 1. Начало адаптации.
- Раунд 3: Ты видел 2 раунда паттернов. Глубокое чтение vs контрчтение.

Это создаёт покерную динамику, где игра становится ИНТЕРЕСНЕЕ с каждым раундом.

Все герои имеют:

### 4.3 Оси баланса

- **Дальность vs Подвижность:** Дальнобойные = медленные (Ястреб: дальность 10, 3 шага). Быстрые = ближнебойные (Тень: 6 шагов, дальность 3).
- **Выживаемость vs Огневая мощь:** Бронированные стреляют реже (Страж: ПЗ 2). Частострелы хрупкие (Берсерк: ПЗ 0, нет Брони).
- **Информация vs Сырая сила:** Инфо-герои (Разведчик, Призрак, Мираж) манипулируют тем, что знает противник.
- **Контроль зон vs Точность:** Контролёры (Подрывник, Инженер) формируют карту. Точечные герои (Лучник, Ястреб) требуют точных предсказаний.

### 4.4 План запуска

- **Альфа (Фаза 1):** 4 героя — Лучник, Танк, Тень, Разведчик
- **Сезон 1 (Фаза 4):** +4 героя — Маг, Подрывник, Страж, Призрак
- **Сезон 2 (Фаза 5):** +4 героя — Инженер, Берсерк, Ястреб, Мираж

## 5. Система карт

### 5.1 Параметры карт

| Параметр | Диапазон | Описание |
|----------|----------|----------|
| Размер | 8×8, 10×10, 12×12 | Разные размеры для разного опыта |
| Стены | 5-30% клеток | Разрушаемые и неразрушаемые |
| Подбираемые предметы | 2-4 на карту | На фиксированных позициях |
| Точки спавна | 2 | Гарантированная минимальная дистанция |
| Симметрия | Ротационная или Зеркальная | Обеспечивает честность |

### 5.2 Типы карт

**Открытое Поле (8×8)** — «Ковбойская дуэль»
- Минимум стен (5-10%). Длинные линии обстрела.
- Выгодно: Лучник, Ястреб. Невыгодно: Берсерк (негде подойти под прикрытием).

**Лабиринт (10×10)** — «Кошки-мышки»
- Много стен (25-30%), множество коридоров и углов.
- Выгодно: Тень, Маг, Призрак. Невыгодно: Берсерк (Рывок блокируется стенами).

**Мосты (10×10)** — «Контроль переправы»
- Открытые зоны, соединённые узкими мостами через пропасти (непроходимые).
- Выгодно: Подрывник (уничтожить мост), Танк (столкнуть с моста).

**Руины (12×12)** — «Война на истощение»
- Большая карта, микс разрушаемых стен и открытых пространств.
- Выгодно: Подрывник (переформатирование карты), Разведчик (быстрое покрытие).

**Арена (8×8)** — «Гладиаторская яма»
- Круглая, стены по краям, открытый центр.
- Выгодно: Берсерк, Танк (форсированный ближний бой). Невыгодно: Лучник, Ястреб.

### 5.3 Подбираемые предметы на карте

| Предмет | Эффект | Визуал |
|---------|--------|--------|
| 🛡️ Осколок Брони | +1 Броня (поглощает одно попадание) | Синий кристалл |
| 👁️ Сфера Разведки | Видишь 1-е действие противника в следующем раунде | Фиолетовый глаз |
| ⚡ Ускорение | +1 Скорость на следующий раунд | Жёлтая молния |
| 🎯 Усиление Дальности | +2 Дальность на следующий раунд | Красный прицел |

**Правила:**
- Можно нести только 1 предмет. Новый заменяет старый.
- Предметы восстанавливаются каждый раунд.
- Позиции ВИДНЫ обоим игрокам.

### 5.4 Сужение карты

```
  Раунд 1: Полная карта.
  Раунд 2: Внешнее кольцо в 1 клетку → ОПАСНАЯ ЗОНА 
            (мгновенное убийство если стоишь там в конце раунда).
  Раунд 3: Внешнее кольцо в 2 клетки → ОПАСНАЯ ЗОНА.
  
  Пример для 10×10:
    Раунд 1: 100 играбельных клеток
    Раунд 2: 64 клетки (внутренние 8×8)
    Раунд 3: 36 клеток (внутренние 6×6)
```

Принуждает к конфронтации. Исключает кемпинг. Гарантирует завершение матчей.

## 6. Система прогрессии (Мета-петля)

### 6.1 Четыре параллельные петли

```
ПЕТЛЯ 1: Мастерство героя  → «Хочу освоить ЭТОГО героя»
ПЕТЛЯ 2: Рейтинг            → «Хочу забраться выше»  
ПЕТЛЯ 3: Сезонный контент   → «Не хочу пропустить это»
ПЕТЛЯ 4: Социал/Клубы       → «Мои друзья рассчитывают на меня»
```

### 6.2 Мастерство героя (Уровень 1-50 для каждого героя)

**Опыт:** Победа +30 XP, Поражение +10 XP, Задания +бонус.

**Вехи (ТОЛЬКО косметика — НИКАКИХ игровых преимуществ):**
- Ур.5: Титул («Новичок-Лучник»)
- Ур.10: Разблокировка Варианта A способности (сайдгрейд)
- Ур.15: Рамка профиля
- Ур.20: Вариант B способности
- Ур.25: Анимированный портрет
- Ур.30: Вариант C способности
- Ур.35: Эффект убийства
- Ур.40: Редкий скин
- Ур.50: УЛЬТИМАТИВНЫЙ СКИН (престижная награда)

**Варианты способностей (пример Лучника):**
- По умолчанию: Рикошет (отскок от 1 стены)
- Вариант A: Ядовитая стрела — без отскока, замедляет цель -1 Скорость на следующий раунд
- Вариант B: Двойной выстрел — 2 снаряда веером, дальность -3
- Вариант C: Снайперский выстрел — дальность +2, но требует 2 шага неподвижности перед стрельбой

**КРИТИЧЕСКИ ВАЖНО:** Варианты = сайдгрейды, НЕ апгрейды. Базовый Рикошет не хуже Снайперского — они для разных ситуаций.

### 6.3 Рейтинговая система

```
  Бронза → Серебро → Золото → Платина → Алмаз → Легенда
  (0 РО)   (1000)    (2000)   (3000)   (4000)   (5000+)
  
  Победа: +25 РО (+5 за серию побед, макс +15 бонуса)
  Поражение: -20 РО
  Сезонный сброс: потеря 30% РО каждые 6 недель
```

### 6.4 Сезоны (6 недель каждый)

Каждый сезон: тема, 1-2 новых героя, новая карта, Боевой Пропуск, эксклюзивная косметика, временный режим.

**Боевой Пропуск:** Бесплатная дорожка (для всех) + Премиум ($5). 30 уровней.

### 6.5 Клубы и социал

- Клубы: 5-30 игроков, чат, таблица лидеров.
- Войны Клубов: раз в две недели, 5v5 индивидуальные матчи.
- Друзья, режим Зрителя, дружеские матчи.

### 8.1 Технологический стек

| Компонент | Технология | Обоснование |

### 8.3 Поток матча (техническая сторона)

```
1. МАТЧМЕЙКИНГ
   Клиент → WS Gateway: 'match:find' (ранг, герой)
   MatchModule: подбор по рангу через MatchmakingService
   WS Gateway → Клиенты: 'match:found' (противник, карта, спавны)

2. ПЛАНИРОВАНИЕ
   Клиент: игрок программирует действия (UI → ActionQueue)
   Клиент → WS Gateway: 'round:commit' { hash: SHA256(actions + nonce) }
   MatchService: ждёт обязательства обоих ИЛИ истечения таймера
   WS Gateway → Клиенты: 'round:reveal-request'
   Клиент → WS Gateway: 'round:reveal' { actions, nonce }
   MatchService: проверяет SHA256(actions + nonce) == hash

3. ИСПОЛНЕНИЕ
   ActionResolver: разрешает все шаги (Фаза1: Движение → Фаза2: Стрельба → Фаза3: Урон)
   WS Gateway → Клиенты: 'round:results' { steps: StepResult[] }
   Клиент: проигрывает 3D анимацию одновременного исполнения

4. ПОСЛЕ РАУНДА
   WS Gateway → Клиенты: 'round:end' { mapShrink, result }
   Если убийство: матч окончен. Если нет: следующий раунд.

5. ПОСЛЕ МАТЧА
   PlayerService: обновляет Мастерство, Рейтинг, Боевой Пропуск
```

### 8.5 WebGL оптимизация

- **IL2CPP:** Code generation = OptimizeSize
- **Managed Stripping Level:** High (минимальный размер билда)
- **Текстуры:** ASTC формат для мобильных браузеров
- **Сжатие:** Brotli для минимального размера загрузки
- **Исключения:** Отключены (None) для производительности
- **WASM 2023:** Включены новые возможности WebAssembly
- **LTO:** Disk Size оптимизация для линковки
- **Data Caching:** Включено для повторных загрузок
- **Целевой FPS:** 60 на мобильных браузерах

### 8.4 Анти-чит

- **Хеш-обязательство:** Клиент отправляет хеш ДО раскрытия. Исключает подглядывание действий противника.
- **Серверная авторитетность:** ВСЁ разрешение на сервере. Клиент = только отображение.
- **Ограничение частоты:** Макс 20 матчей/час. CAPTCHA для ставок. Детекция ботов.
